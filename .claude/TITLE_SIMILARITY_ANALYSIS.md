## 🎯 発見されたパターン（タイトル共通構造）

### 1. **統一された構文テンプレート**
```
【パターンA】「✗✗✗」の正体：『△△△』が▲▲▲を告げる◆◆◆
【パターンB】「✗✗✗」と▲▲▲：△△△が物理的◆◆◆に衝突する
【パターンC】なぜ「✗✗✗」は◆◆◆なのか？『△△△』と▲▲▲の逆転
【パターンD】「✗✗✗」が終わる日：▲▲▲が告げた△△△と◆◆◆の衝突
```

**結論**: 7本のタイトルすべてが、「質問形式」または「コロン区切り形式」のいずれかを採用しており、バリエーションがない。

### 2. **繰り返される「同じ単語セット」**
- 📍 **「物理的」**：16, 15, 11日のタイトルで登場 ✓
- 📍 **「構造」「構造的」**：17, 14日で登場 ✓
- 📍 **「衝突」「転換」**：16, 15日で登場 ✓
- 📍 **「正体」「の正体は」**：14, 11日で登場 ✓
- 📍 **「〇〇が告げる」「告げた」**：17, 16日で登場 ✓
- 📍 **「二度と戻ってこない」「逆転」**：14, 13日で登場 ✓

**結論**: 異なるテーマのニュースなのに、**LLMが同じ語彙・同じ論理構造を再生成している**。

### 3. **プロンプトレベルの根本原因**

#### ❌ **原因1: `outline`プロンプト（L325-359）**
```yaml
**タイトル生成の自由度:**
タイトルは視聴者が関心を持つ角度から、自由かつ創意的に生成してください。
過去の事例に拘束されず、ニュースの本質を最も効果的に伝える表現を選択すること。
```

**問題点**:
- 「過去の事例に拘束されず」と言いながら、実際には **「過去のタイトル一覧」が提供されていない**
- LLMは「視聴者ウケする一般的なパターン」を推測する → **結果的に同じテンプレートを再利用**

#### ❌ **原因2: `memory`システムプロンプト（L444-457）**
```yaml
あなたは特化型の知識抽出エージェントです。
あなたの目標は、動画台本の「本質（essence）」を、将来の検索に備えた構造化された形式で抽出することです。
```

**問題点**:
- 過去の「essence」を格納する設計だが、**新規タイトル生成時にそれを「制約」として使用しない**
- 結果：LLMが無意識に「既に使った表現」を再利用してしまう

#### ❌ **原因3: `content`の`one_shot`プロンプト（L278-323）**
```yaml
タイトルは「【緊急】」「正体」等の煽り表現を排し、内容の深さを想起させる知的なものにする。
```

**問題点**:
- 「煽り表現を排す」と言いながら、実際には **「〇〇の正体」「△△が告げる」といったテンプレート的な表現を促している**
- 「知的なもの」という曖昧な指示が、結果的に「同じ知的表現パターン」の繰り返しを招く

## 📋 根本原因: プロンプト設計の欠陥

### **真の問題：「多様性制御がない」**

現在のプロンプトは：
1. ✅ 「個別のニュース」を入力として受け取る
2. ✅ 「知的で誠実」というトーン指示はある
3. ❌ **「過去7日間のタイトルリストを見て、それと被らない表現を選択する」という制約がない**
4. ❌ **「タイトルテンプレートの多様性」を測定・強制する仕組みがない**

### **なぜ同じパターンが出現するのか？**
```
LLM（Claude）の思考プロセス：
  「知的で誠実」
  + 「視聴者がクリックしたくなる」
  + 「マクロ経済テーマ」
  = 統計的に「コロン区切り形式」「物理的✗構造」というパターンに偏る

結果：言語モデルの「確率分布」が特定のテンプレートに集中 → 毎日同じ構造
```

## ✅ **解決策**

### **1. 過去タイトル一覧を`content`プロンプトに **明示的に** 含める**
```yaml
【直近7日間のタイトル（これらは避けること）】
- 金利の時代から「歩留まり」の時代へ：...
- 「金利の支配」が終わる日：...
... (全7件)

【タイトル多様性チェックリスト】
以下の構文は既に使用済みなため、異なる構文を選択してください：
✗ 「XXX」の正体：△△△が▲▲▲を告げる
✗ 「XXX」と△△△：▲▲▲が物理的◆◆◆に衝突
✗ なぜ「XXX」は◆◆◆なのか？『△△△』と▲▲▲の逆転
```

### **2. タイトルパターンの「多様性テンプレート」を明示**
```yaml
【許可される複数のタイトル構造】
1. 疑問形：「〇〇で△△が起きている理由」
2. 衝突型：「〇〇と△△の本当の関係」
3. 発見型：「新たなデータが示す〇〇の現実」
4. 予測型：「〇〇は今後6ヶ月でこう変わる」
5. 対比型：「〇〇という常識が廃れるワケ」
6. インパクト型：「ここから生まれる新しい投資機会」
```

### **3. プロンプトに「タイトル生成ロジック」を追加**
```yaml
【タイトル生成ルール】
- 「物理的」「構造」「衝突」「転換」の使用回数：過去7日で各1回まで
- コロン（：）の使用数：前週と異なる個数を選択
- 開きカッコ「「」の使用個数：毎日異なるバリエーション

【出力チェック】
生成したタイトルが以下のいずれかに該当する場合、破棄して再生成：
- 過去7日のタイトルと単語が3個以上重複している
- 同じ構文パターンを2回以上使用している
- 同じ修飾語「〜的」が連続している
```

### **4. ワークフロー改修（short-term）**
```yaml
# Taskfile.ymlに追加
task title:check-diversity
  - bun run scripts/check_title_diversity.ts --recent-days 7 --output diversity_report.json
```
