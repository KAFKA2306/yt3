# Research段階のテーマ多様性強制 - 修正完了

## ✅ 修正内容（config/default.yaml）

### 1. **5つのニュースカテゴリーを明示的に定義**
```yaml
【ニュースカテゴリーの定義】
1. マクロ経済（金利・インフレ・GDP・雇用統計・中央銀行政策）
2. 企業財務（決算・M&A・投資計画・リストラ・配当）
3. 地政学（国際関係・戦争・制裁・貿易戦争・同盟）
4. テクノロジー（新製品・AI・半導体・特許・イノベーション）
5. 市場心理（投資家行動・ボラティリティ・トレンド・市場不安）
```

### 2. **選定ルール：「異なる2つのカテゴリーから各1つ」**
```yaml
# 変更前：
2. 収集したニュースの中から、市場や社会に最も大きな直接的影響（インパクト）を与えている事象を、
   主観的な解釈を挟まずに3つ選定してください。

# 変更後：
2. 収集したニュースを上記5カテゴリーに分類してください。
3. 異なる2つのカテゴリーから、最もインパクトが大きい事象を各1つ選定してください。
   （例：マクロ経済1つ＋テクノロジー1つ、など）
4. 過去のテーマリストが提供される場合、同じカテゴリーが3日連続しないよう配慮してください。
```

### 3. **出力形式：カテゴリー追跡**
```json
{
  "selected_topics": [
    {
      "category": "マクロ経済",  // ← 新フィールド：カテゴリーを明記
      "selected_topic": "...",
      "reason": "...",
      "angle": "...",
      ...
    },
    {
      "category": "テクノロジー",
      ...
    }
  ]
}
```

### 4. **user_template：過去テーマ一覧を追加**
```yaml
【過去7日間のテーマ一覧（カテゴリー多様性を意識してください）】
{recent_themes}
```

## 🔧 **コード側の実装が必要**

### **必要な処理：**

1. **`fetchRecentThemes()`関数を実装**
   - 過去7日の `research/output.yaml` から `selected_topics[].category` を抽出
   - 日付順でフォーマット（例：`2026-03-17: マクロ経済、テクノロジー`）

2. **research実行前に過去テーマを取得**
   ```typescript
   const recentThemes = await fetchRecentThemes(7);
   const researchPrompt = userTemplate.replace(
     '{recent_themes}',
     recentThemes
   );
   ```

3. **research出力の `selected_topics` が「異なる2カテゴリー」であることを検証**
   ```typescript
   const categories = output.selected_topics.map(t => t.category);
   if (categories.length !== 2 || categories[0] === categories[1]) {
     throw new Error('Must select from 2 different categories');
   }
   ```
